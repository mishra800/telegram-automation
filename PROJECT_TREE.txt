================================================================================
DATA FLOW
================================================================================

1. CONTENT CREATION:
   Scheduler → ContentGenerator → ImageGenerator → TelegramBot → Database

2. ANALYTICS:
   Collector → TelegramBot (get stats) → Database (update)

3. STRATEGY:
   Analyzer → Database (query) → Calculate weights → Database (update)

4. DASHBOARD:
   Flask → Database (query) → Matplotlib (chart) → HTML (display)

================================================================================
EXECUTION FLOW
================================================================================

START:
  python main.py
    ├── Load config from .env
    ├── Validate configuration
    ├── Initialize database
    ├── Start scheduler
    │   ├── Schedule content posts
    │   ├── Schedule analytics collection
    │   └── Schedule strategy adjustment
    ├── Start dashboard (Flask)
    └── Wait for Ctrl+C

POST (Scheduled or Manual):
  Scheduler.create_and_post()
    ├── Select topic (weighted random)
    ├── Generate content (Ollama)
    ├── Generate image (SD or gradient)
    ├── Post to Telegram
    └── Save to database

ANALYTICS (Daily at 02:00):
  Collector.collect_stats()
    ├── Get posts older than 24h
    ├── Fetch stats from Telegram
    └── Update database

STRATEGY (Weekly Monday 03:00):
  Collector.analyze_and_adjust()
    ├── Calculate 7-day performance
    ├── Compute engagement scores
    └── Update topic weights

DASHBOARD (Continuous):
  Flask app at localhost:5000
    ├── Serve HTML interface
    ├── API: /api/stats
    ├── API: /api/topic-performance
    └── API: /api/engagement-chart
